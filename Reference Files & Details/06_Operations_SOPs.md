## 📖 Story Summary: คู่มือที่ทำให้ทุกอย่างเป็นระบบ

### ทำไมต้องมี SOPs?

ลองนึกภาพว่าคุณเพิ่งผลิต E-book เสร็จ ตื่นเต้นมากอยากลงขายทันที แต่พอเริ่มทำก็พบว่า... เอ๊ะ ต้องทำภาพอะไรบ้างนะ? ราคาตั้งเท่าไหร่ดี? ของแถมมีอะไรบ้าง? ลิงก์ดาวน์โหลดทำยังไง? สุดท้ายเสียเวลาไปครึ่งวันแค่หาข้อมูลที่ควรจะมีอยู่แล้ว

**SOPs (Standard Operating Procedures)** คือ "สูตรสำเร็จ" ที่ทำให้ไม่ต้องคิดใหม่ทุกครั้ง ทำตาม Checklist ก็เสร็จ ไม่ตกหล่น ไม่สับสน แม้จะเหนื่อยหรือมีงานเยอะก็ยังทำได้ถูกต้อง

### Product Launch Checklist: 4 เฟสสู่การลงขาย

ทุกครั้งที่จะลงขายสินค้าใหม่ ที่พักใจใช้ **4-Phase Workflow** ที่ครอบคลุมทุกขั้นตอน

**Phase 1: Production & Value Stack** คือการเตรียม "ของ" ที่จะส่งมอบให้ลูกค้า สำหรับ E-book ต้องมี Main E-book (PDF Interactive), Interactive Workbook (Bonus), Wallpaper Set 5 รูป (Bonus), Free Sample (สำหรับตัวอย่างก่อนซื้อ), และ Delivery PDF (Welcome Guide พร้อมลิงก์ดาวน์โหลด) ทุกอย่างต้องเสร็จก่อนลงขาย

**Phase 2: Visual Marketing** คือการเตรียม "หน้าตา" ของหน้าร้าน ต้องมี 5 ภาพโปรโมท: Cover (Hero Shot) ปกสินค้าชัดๆ, Inside Look โชว์เนื้อหาภายใน, Problem & Solution แสดง Before/After, Cozy Lifestyle บรรยากาศการใช้งาน, และ What You Get สรุปของที่ได้ทั้งหมด

**Phase 3: Listing Details** คือการเตรียม "บทพูดขายของ" ประกอบด้วย ชื่อสินค้าที่ชัดเจนจำง่าย, คำโปรยสั้นที่หยุดนิ้วโป้ง ("อ่านจบใน 5 นาที สำหรับวันที่ใจพัง"), รายละเอียด 1,000 ตัวอักษร (Story + Pain + Solution), จุดขาย 6 ข้อ, Category, และ Target Audience

**Phase 4: File Organization** คือการจัดระเบียบไฟล์สำหรับส่งมอบ มี Folder Structure มาตรฐาน: 01_Main_Product, 02_Workbook, 03_Wallpapers, 04_Free_Sample, 05_Marketing อัปโหลดขึ้น Google Drive ตั้ง Permission ให้ Anyone with link สามารถ View ได้

### Welcome PDF System: จดหมายต้อนรับที่อบอุ่น

ทุกครั้งที่ลูกค้าซื้อสินค้า พวกเขาจะได้รับ **Welcome PDF** — ไฟล์ 1 หน้าที่มีลิงก์ดาวน์โหลดและข้อความต้อนรับ

**Layout มาตรฐาน** ประกอบด้วย: Header (Logo + "ยินดีต้อนรับกลับบ้านนะ..."), Greeting ("ขอบคุณที่อนุญาตให้ตัวเองได้พัก... ในวันที่โลกบอกให้รีบวิ่ง"), Hero Section (Product Mockup + รายการสิ่งที่ได้รับ), Action Area (ปุ่ม Download + QR Code), Instruction (วิธีเปิดไฟล์ iOS/Android), และ Footer ("ถ้าโหลดไม่ได้ ทักหาฟ้าใสได้ตลอดเลยนะ")

มี **Canva Workflow** สำหรับสร้าง Welcome PDF ใหม่ทุกครั้ง: สร้าง Master Template → Duplicate Page → Swap Content (รูป, ชื่อ, ลิงก์) → Export เป็น PDF Standard

### Content Production Workflow: สายการผลิตเนื้อหา

ทุกเนื้อหาที่สร้างต้องผ่าน **4 ขั้นตอน**:

**INPUT** — รับ Brief สินค้า/แคมเปญ, Theme ประจำเดือน, Content Calendar

**GENERATE** — เขียน Copy (ใช้ Template), สร้าง Visual (ใช้ AI Prompt), ประกอบร่างใน Canva

**REVIEW** — ตรวจสอบ Brand Voice, ตรวจสอบ Visual Identity, Approve หรือ Request Changes

**PUBLISH** — Schedule ตามช่อง, Cross-post ถ้าเหมาะ, Monitor Engagement

เอกสารนี้รวมเวลาโดยประมาณ: Quote Card ใช้เวลา 15-30 นาที, Carousel 5 slides ใช้เวลา 1-2 ชั่วโมง, Short Video ใช้เวลา 2-3 ชั่วโมง, Blog Article ใช้เวลา 3-5 ชั่วโมง, E-book Chapter ใช้เวลา 4-6 ชั่วโมง

### UCE Prompt Library: คลัง Prompts สำหรับทุกช่อง

**Master Visual Code** กำหนด Default Settings: Primary Color = Deep Teal #0D7377, Secondary = Sage Green #9DC08B, Background = Cream #FFF8E7, Accent = Warm Amber #E6A23C, Font Thai = Sans-serif ไม่มีหัว, Font English = Inter, Outfit, Poppins

มี **Channel-Specific Prompts** สำหรับ Facebook Carousel, Instagram Reels Cover, TikTok Video, Pinterest Pin, และ LINE Broadcast

### Customer Service SOPs: วิธีดูแลลูกค้า

เอกสารนี้รวม SOPs สำหรับปัญหาที่พบบ่อย:

**Payment Issues** — ลูกค้าจ่ายเงินแล้วแต่ไม่ได้รับ Email: ขอ Email ที่ใช้สั่งซื้อ → ตรวจสอบ Stripe Dashboard → ตรวจสอบ Database → ถ้าพบ Order ให้ Resend Email, ถ้าไม่พบให้ขอหลักฐานการโอน

**Refund Requests** — นโยบาย: ไม่รับคืนเงินหลังดาวน์โหลด ยกเว้นไฟล์เสียหรือส่งผิดสินค้า ภายใน 7 วัน ถ้ายังไม่ Download สามารถ Refund ได้

**Download Issues** — ปัญหาที่พบบ่อย: ลิงก์หมดอายุ, เปิดไฟล์ไม่ได้, ไฟล์เสีย ถาม iPhone/Android แล้วให้คำแนะนำเฉพาะ ถ้ายังไม่ได้ให้ Generate ลิงก์ใหม่

### Audio Production SOPs: สร้างเสียงบำบัด

สำหรับ Healing Audio Series มี **Suno Prompt Templates** สำหรับทุก Room:

**Focus Mode (BPM 60-70):** "Lofi jazz piano, soft brushed drums at 65 BPM, warm analog synth pads, gentle rain foley in background. Relaxed but focused mood. No lyrics. A=432Hz tuning preferred."

**Deep Sleep (BPM 40-50):** "Drone ambient, binaural-friendly low frequencies, subtle nature sounds (distant ocean, soft rain). Extremely slow tempo at 45 BPM. Dark, deep, hypnotic. A=432Hz tuning. No melody, pure texture."

มี **Checklist** สำหรับ Production: Generate base track → Review and select → Edit (trim, normalize) → Add foley layer → Check BPM → Export MP3/WAV → Tag metadata → Test on phone speaker + headphones

### Asset Management: จัดระเบียบไฟล์

ทุกไฟล์ต้องผ่าน **4 ขั้นตอน**: RAW (AI Output, Drafts) → RENAME (Consistent Naming) → VAULT (Organized Archive) → EXPORT (Ready to Publish)

**Naming Convention:** `[TYPE]_[PRODUCT]_[VARIANT]_[DATE].[EXT]` เช่น `IMG_GentlePause_Cover_20241215.png` หรือ `AUD_FocusMode_01_60min.mp3`

**Folder Structure** สำหรับ Master Vault: 01_Brand (Logo, Colors, Fonts), 02_Products (E-books, Planners, Audio, Wallpapers, Oracle), 03_Marketing (Social_Posts, Email_Assets, Ads), 04_Templates (Canva, Prompts, Scripts), 05_Archive ([YYYY-MM])

### บทสรุป

เอกสาร Operations & SOPs นี้คือ "คู่มือปฏิบัติงาน" ที่ทำให้ทุกอย่างเป็นระบบ ตั้งแต่ Product Launch 4 Phases, Visual Prompts, Welcome PDF System, Content Production, Customer Service, Audio Production, ไปจนถึง Asset Management ใช้เป็นเอกสารอ้างอิงทุกครั้งที่ต้องทำงาน เพื่อให้ได้ผลลัพธ์ที่สม่ำเสมอและมีคุณภาพ **ทำตาม Checklist ก็เสร็จ ไม่ต้องคิดใหม่ทุกครั้ง**

**ชื่ออย่างเป็นทางการ:** บันทึกฝึกเบา (The Lightness Practice)

| Element | Specification |
|---------|---------------|
| **Concept** | "พื้นที่ปลอดภัยบนกระดาษ" (Paper Sanctuary) ไม่เน้นความสวยงาม แต่เน้นการระบายและทำความเข้าใจตัวเอง |
| **Format** | PDF ขนาด A4 (เหมาะสำหรับ Print เขียนมือ หรือเปิดเขียนใน GoodNotes บน iPad) |
| **Total Pages** | 8 หน้า (รวมปก) |

### 2.1 Page Breakdown (รายละเอียดภายในเล่ม)

| Page | ชื่อหน้า | Goal | Content |
|------|---------|------|---------|
| **1** | ปกหน้า (Cover) | - | ลายเส้นขนนกวางบนหิน (Minimalist), พื้นที่ว่างเยอะๆ สีครีมสะอาดตา |
| **2** | วิธีใช้ใจกับสมุดเล่มนี้ (User Guide) | ปรับ Mindset | กฎ 3 ข้อ: อนุญาตให้ไม่สวย, อนุญาตให้ข้าม, อนุญาตให้ระบาย |
| **3** | ตาชั่งทางใจ (The Emotional Scale) | Diagnosis | Checklist อาการทางกาย/ใจ + สเกล 1-10 |
| **4** | กระดานเทขยะ (The Brain Dump) | Unload | ตาราง 3 ช่อง: To-do / Worries / Let Go |
| **5** | บันทึกการหายใจ (Breathing Tracker) | Habit Building | ตาราง 7 วัน สำหรับเทคนิค 4-7-8 |
| **6** | ดีท็อกซ์ความคาดหวัง (Expectation Detox) | Deep Healing | ตารางเปรียบเทียบ "เสียงคนอื่น" vs "เสียงหัวใจ" |
| **7** | สคริปต์ปฏิเสธฉบับฉัน (My 'No' Script) | Skill Building | พื้นที่เขียนประโยคปฏิเสธ + ตัวอย่างไกด์ |
| **8** | จดหมายถึงตัวเอง (Letter to Self) | Self-Compassion | หน้าบรรทัดพร้อมคำขึ้นต้น "ถึง ตัวฉันในวันที่โลกใจร้าย..." |

### 2.2 Cover Specifications

```
Visual: ลายเส้นขนนกวางบนหิน (Minimalist) พื้นที่ว่างเยอะๆ สีครีมสะอาดตา
Text: ชื่อ "บันทึกฝึกเบา" และ Subtitle "สมุดพกพาสำหรับวางหัวใจลงชั่วคราว"
```

### 2.3 Page Details (แบบฝึกหัดแต่ละหน้า)

**Page 3: ตาชั่งทางใจ (The Emotional Scale)**
- Goal: วินิจฉัย (Diagnosis) ความเครียดที่ซ่อนอยู่
- Action: 
  - มี Checklist อาการทางกาย/ใจ (เช่น กัดฟัน, นอนไม่หลับ) ให้ติ๊ก
  - มีสเกล 1-10 ให้วงกลมระดับความหนัก

**Page 4: กระดานเทขยะ (The Brain Dump)**
- Goal: เทออก (Unload) สิ่งที่วนในหัวก่อนนอน
- Action: ตาราง 3 ช่อง:
  - สิ่งที่ต้องทำ (To-do)
  - สิ่งที่กังวล (Worries)
  - สิ่งที่คุมไม่ได้ (Let Go)

**Page 5: บันทึกการหายใจ (Breathing Tracker)**
- Goal: สร้างนิสัย (Habit Building) การกลับมาอยู่กับลมหายใจ
- Action: ตาราง 7 วัน ให้ระบายสีวงกลมเล็กๆ เมื่อฝึกหายใจ (4-7-8) ครบ 1 นาที

**Page 6: ดีท็อกซ์ความคาดหวัง (Expectation Detox)**
- Goal: เยียวยาปม (Deep Healing) เรื่องความกตัญญู/หน้าตา
- Action: ตารางเปรียบเทียบซ้าย-ขวา:
  - "เสียงคนอื่นบอกให้ทำ" vs "เสียงหัวใจฉันต้องการ"

**Page 7: สคริปต์ปฏิเสธฉบับฉัน (My 'No' Script)**
- Goal: ฝึกทักษะ (Skill) การสร้างขอบเขต (Boundaries)
- Action: พื้นที่ว่างในกรอบคำพูด ให้ลองเขียนประโยคปฏิเสธที่ตัวเองพูดแล้วสบายใจ (มีตัวอย่างไกด์ให้)

**Page 8: จดหมายถึงตัวเอง (Letter to Self)**
- Goal: ใจดีกับตัวเอง (Self-Compassion)
- Action: หน้ากระดาษลายเส้นบรรทัด พร้อมคำขึ้นต้น "ถึง ตัวฉันในวันที่โลกใจร้าย..." ให้เขียนให้กำลังใจตัวเอง

## 3. Wallpaper Set Template

### 3.1 Wallpaper Details (รายละเอียดทั้ง 5 รูป)

| # | ชื่อ | Visual | Quote | Purpose |
|---|-----|--------|-------|---------|
| **WP 01** | การกลับมา (Grounding) | หินสีเทาเข้มเรียงซ้อนกันอย่างสมดุลกลางน้ำนิ่ง (Zen Stones) โทนสี Deep Teal | "กลับมาอยู่ตรงนี้... ที่ที่มีแค่ลมหายใจ" | เรียกสติเวลาฟุ้งซ่าน หรือ Panic |
| **WP 02** | การอนุญาต (Permission) | ดอกหญ้า/ปุยฝ้าย สีขาวฟูๆ ลอยล่องไปตามลม โทนสี Cream/Sage Green | "อนุญาตให้ตัวเองวาง... ในวันที่ถือไม่ไหว" | ลดความรู้สึกผิดเวลาพักผ่อน |
| **WP 03** | การยอมรับ (Self-Acceptance) | ถ้วยเซรามิกที่มีรอยร้าวแต่ซ่อมด้วยทอง (Kintsugi) โทนสีเข้มตัดกับเส้นทอง | "ไม่สมบูรณ์แบบ... แต่สมบูรณ์พร้อมสำหรับการถูกรัก" | เยียวยา Imposter Syndrome |
| **WP 04** | ขอบเขต (Boundaries) | กระท่อมเล็กๆ ที่มีไฟอุ่นๆ อยู่กลางป่าใหญ่ที่มืดมิด (Safe Space) | "ขอบเขตที่ชัดเจน คือการบอกรักตัวเองที่ดังที่สุด" | เตือนใจสำหรับคนขี้เกรงใจ (People Pleaser) |
| **WP 05** | ความหวัง (Hope) | แสงอาทิตย์ยามเช้าที่ส่องทะลุหมอกและทิวเขา โทนสีอุ่นทอง (Warm Gold) | "พรุ่งนี้... ค่อยเริ่มใหม่ก็ยังทัน" | ให้กำลังใจในวันที่รู้สึกพัง หรือ Burnout |

### 3.2 Value Stack Summary

> **สรุปสิ่งที่ลูกค้าจะได้รับ:** เมื่อลูกค้าโหลดไฟล์ เขาจะได้ Folder ที่มี:

## 9. Customer Service SOPs

### 9.1 Payment Issues

**สถานการณ์: ลูกค้าจ่ายเงินแล้วแต่ไม่ได้รับ Email**

**ขั้นตอน:**
1. ขอ Email ที่ใช้สั่งซื้อ
2. ตรวจสอบ Stripe Dashboard → เช็ค Payment Status
3. ตรวจสอบ Database → เช็ค Order Record
4. **ถ้าพบ Order:**
   - Resend Email ด้วย Download Link
   - แจ้งลูกค้าให้เช็ค Spam folder
5. **ถ้าไม่พบ Order:**
   - ขอหลักฐานการโอน
   - ตรวจสอบกับ Stripe เพิ่มเติม
   - ถ้ายืนยันได้ → สร้าง Order ใหม่และส่งไฟล์

**Script ตอบลูกค้า:**
```
สวัสดีนะ 💚

ขอโทษที่ลำบากนะ เดี๋ยวเราช่วยดูให้เลย

ช่วยแจ้ง Email ที่ใช้สั่งซื้อให้หน่อยได้ไหม?
แล้วเดี๋ยวเราจะส่งลิงก์ดาวน์โหลดให้ใหม่เลย

ฟ้าใส
```

### 9.2 Refund Requests

**นโยบาย Refund:**
- Digital Products: ไม่รับคืนเงินหลังดาวน์โหลด
- ยกเว้น: ไฟล์เสีย, ส่งผิดสินค้า
- ระยะเวลา: ภายใน 7 วันหลังซื้อ

**ขั้นตอน:**
1. ถามเหตุผลที่ต้องการ Refund (ด้วยความเข้าใจ)
2. ตรวจสอบว่า Download แล้วหรือยัง
3. **ถ้ายังไม่ Download:**
   - Issue Refund ผ่าน Stripe
   - แจ้งลูกค้า
4. **ถ้า Download แล้ว:**
   - อธิบายนโยบาย
   - เสนอทางเลือก: แลกเป็นสินค้าอื่น / Credit

**Script ตอบลูกค้า (กรณีอนุมัติ):**
```
สวัสดีนะ 💚

เข้าใจเลย ไม่มีปัญหานะ
เราจะทำ Refund ให้ภายใน 3-5 วันทำการ

ถ้ามีโอกาสกลับมาเมื่อไหร่ เรายินดีต้อนรับเสมอนะ

ฟ้าใส
```

### 9.3 Download Issues

**ปัญหาที่พบบ่อย:**
1. ลิงก์หมดอายุ
2. เปิดไฟล์ไม่ได้
3. ไฟล์เสีย

**ขั้นตอน:**
1. ถาม: iPhone หรือ Android? เกิดอะไรขึ้นเมื่อกด?
2. ส่งคำแนะนำเฉพาะ:
   - **iOS:** แนะนำเปิดใน Safari → Save to Files / GoodNotes
   - **Android:** แนะนำใช้ Xodo / Adobe Acrobat
3. ถ้ายังไม่ได้ → Generate ลิงก์ใหม่และส่งให้

Drone ambient, binaural-friendly low frequencies, subtle
